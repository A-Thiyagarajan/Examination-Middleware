About Project

From scanned answer sheets to Moodle-ready submissions

Examination Middleware System solves a practical campus problem: institutions often scan physical exam papers, but Moodle expects structured digital submissions. This project acts as the operational bridge between those two worlds.

Smart Upload Processing

Staff upload scanned files with intelligent parsing, validation, and metadata extraction for seamless artifact creation.

Student Verification

Students access their own papers through secure sessions, verify content, and report any mismatches before submission.

LMS Integration

Automated Moodle assignment mapping and submission workflow ensures accurate delivery with full audit traceability.

Features

Built around academic operations, not generic file upload forms

The backend and UI layers reflect real exam-cell workflows, student verification needs, and LMS integration constraints.

Staff Upload Portal

Supports single and bulk uploads, filename validation, exam-session tagging, and centralized artifact tracking for staff teams.

Student Dashboard & Submission

Students see pending and submitted papers, preview answer sheets, report mismatches, and submit only their own artifacts.

Moodle Integration

Maps subject codes to Moodle assignments and executes the save-and-submit flow required for LMS delivery.

Authentication & Security

Uses JWT for staff access, encrypted student LMS tokens, strict ownership checks, and session validation for protected routes.

Audit Logging

Uploads, views, report actions, failed submissions, and workflow transitions are tracked for traceability and support.

Retry & Recovery Design

Artifacts can move through queued or failed states, supporting retry logic and safer handling when LMS services are unstable.

Architecture

Operational flow of the Examination Middleware System

The architecture centers on artifact creation, secure student access, and controlled Moodle submission rather than direct file drops into LMS.
